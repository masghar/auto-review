STEP 1 — BUILD THE EXTENSION ZIP
────────────────────────────────────────────────────────────────────

Include ONLY these files in the ZIP:

  ✓ manifest.json
  ✓ background.js
  ✓ content.js
  ✓ content.css
  ✓ popup.html
  ✓ LICENSE
  ✓ icons/icon16.png
  ✓ icons/icon48.png
  ✓ icons/icon128.png

DO NOT include:
  ✗ store/          (publishing assets, not part of extension)
  ✗ .git/           (version control)
  ✗ CLAUDE.md       (internal dev notes)
  ✗ README.md       (optional — it does no harm but adds no value)
  ✗ .DS_Store       (macOS metadata)
  ✗ node_modules/   (not applicable — no build step)

Zip command (run from the project root):
  zip -r gmb-review-ai-v6.1.0.zip manifest.json background.js content.js content.css popup.html LICENSE icons/

Verify zip contents:
  unzip -l gmb-review-ai-v6.1.0.zip

STEP 6 — PRIVACY PRACTICES TAB
────────────────────────────────────────────────────────────────────

Single purpose statement (enter exactly):
  "Generates and posts AI-written replies to Google Business Profile
   reviews using the Gemini API, using the user's own API key."

Privacy policy URL:
  → You must host store/privacy-policy/privacy-policy.html at a
    public URL first. Options:
    a) GitHub Pages: push to a public repo → enable Pages
    b) Any static host (Netlify, Vercel, etc.) — free tier works
    c) Your own website
  → Enter the full public URL here (e.g. https://yourname.github.io/gmb-review-ai/privacy)

Data handling — check these boxes:
  Personal communications: NO
  Health info: NO
  Financial info: NO
  Authentication info: NO  (API key is not a login credential)
  Personal identifiable info: NO
  Web browsing history: NO
  User activity: NO
  Website content: NO

Host permissions justification (copy this exactly):

  business.google.com:
  "Required to inject the review management side panel into the user's
   Google Business Profile reviews page (business.google.com/reviews).
   The extension reads review text from this page and posts replies
   using the user's existing session."

  generativelanguage.googleapis.com:
  "Required to call Google's Gemini API from the extension background
   service worker using the user's own Gemini API key. This is needed
   because browser CORS rules require an explicit host permission for
   fetch() calls from extension service workers."


────────────────────────────────────────────────────────────────────
STEP 7 — SUBMIT FOR REVIEW
────────────────────────────────────────────────────────────────────

□ All fields on the Store Listing tab are complete
□ At least 1 screenshot uploaded (1280×800 PNG)
□ Small promo tile uploaded (440×280 PNG)
□ Privacy policy URL entered and page is live/accessible
□ Host permission justifications filled in
□ Extension ZIP uploaded without errors

Click "Submit for Review".

Review timeline:
  • First submission: typically 3–7 business days
  • Updates to published extensions: typically 1–3 business days
  • You'll receive an email at asghar.devops@gmail.com when approved
    or if Google requests additional information

If rejected, the rejection email will specify the exact policy violated.
Common first-rejection reasons:
  - Privacy policy URL not accessible
  - Host permissions not justified
  - Description contains keyword stuffing (avoid repeating the same
    phrase more than 2–3 times)

POST-PUBLISH
────────────────────────────────────────────────────────────────────

After the extension goes live:

□ Share the store URL with users (copy from published listing page)
□ Add the store URL to README.md
□ Link to privacy policy from README.md
□ Link to how-to-use guide from README.md
□ Consider adding a "How-to-use" link inside the extension popup

For future updates:
  1. Increment version in manifest.json (e.g. 6.1.0 → 6.2.0)
  2. Update the version comment in content.js line 3
  3. Rebuild ZIP (same files as above)
  4. Upload new ZIP via Developer Dashboard → "Package" tab → "Upload new package"
  5. Resubmit — update review is faster (1–3 days)
